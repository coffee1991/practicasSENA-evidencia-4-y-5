Comandos Básicos de Git ✨ Local y Remoto

Comando 💻 Tipo (Local/Remoto) 📍 Descripción 📝
git init Local 🌱 Inicializa un nuevo repositorio Git local en la carpeta actual. ¡El primer paso!
git status Local 📊 Muestra el estado de los archivos en tu área de trabajo y staging. Te dice qué ha cambiado.
git add <archivo>
o git add .		 Local 👍 Añade cambios de un archivo específico o de todos los archivos nuevos/modificados al área de staging para el próximo commit.
git commit -m "mensaje" Local 💾 Guarda los cambios que están en el área de staging en el historial del repositorio con un mensaje descriptivo.
git log Local 📜 Muestra el historial de commits de la rama actual. Puedes ver quién hizo qué y cuándo.
git branch <nombre> Local 🌿 Crea una nueva rama con el nombre especificado. Útil para trabajar en nuevas funcionalidades aisladamente.
git switch <nombre_rama>
o git checkout <nombre_rama>		 Local 🔄 Cambia a la rama especificada. Te permite moverte entre las diferentes líneas de desarrollo.
git clone <url> Remoto 🌎➡️💻 Clona un repositorio existente desde una URL remota a tu máquina local. ¡Obtienes una copia completa!
git remote add origin <url> Remoto 🔗 Añade una conexión a un repositorio remoto. 'origin' es un nombre común para el repositorio principal.
git remote -v Remoto 📡 Muestra las URL de los repositorios remotos configurados para tu repositorio local.
git fetch origin Remoto 📥 Descarga los objetos y referencias del repositorio remoto ('origin' en este caso), pero no los integra automáticamente en tus ramas locales.
git pull origin <rama> Remoto 🤝 Descarga los cambios del repositorio remoto ('origin', rama especificada) Y los integra automáticamente en tu rama local actual (fetch + merge/rebase).
git push origin <rama> Remoto 💻➡️🌎 Sube tus commits locales a la rama especificada en el repositorio remoto ('origin'). ¡Comparte tu trabajo!
